==================================================
Main Goal
==================================================

1. Filter only 2022 transactions
2. Calculate total spend per product
3. Rank products inside each category
4. Keep top 2 products

==================================================
Step-by-Step Thinking
==================================================

Step 1 — Filter 2022 Data

WHERE EXTRACT(YEAR FROM transaction_date) = 2022

Keeps only transactions from year 2022.

--------------------------------------------------

Step 2 — Group Products

GROUP BY category, product

Creates grouped rows like:

| category    | product          |
|-------------|------------------|
| appliance   | refrigerator     |
| electronics | vacuum           |

--------------------------------------------------

Step 3 — Calculate Total Spend

SUM(spend)

Calculates total revenue generated by each product.

Example:

| category    | product          | total_spend |
|-------------|------------------|--------------|
| electronics | vacuum           | 341          |
| electronics | wireless headset | 249.9        |

--------------------------------------------------

Step 4 — Window Function Ranking

ROW_NUMBER() OVER(
    PARTITION BY category
    ORDER BY SUM(spend) DESC
)

Meaning:

- PARTITION BY category
  → separate ranking inside each category

- ORDER BY SUM(spend) DESC
  → highest spending product gets rank 1

Example:

electronics category:

| product          | total_spend | rn |
|------------------|--------------|----|
| vacuum           | 341          | 1  |
| wireless headset | 249.9        | 2  |

--------------------------------------------------

Step 5 — Keep Top 2

WHERE rn <= 2

Returns only top 2 ranked products per category.

==================================================
Why SUM(spend) Works Inside Window Function
==================================================

This works:

ROW_NUMBER() OVER(
    ORDER BY SUM(spend) DESC
)

because:
- GROUP BY already formed groups
- SUM(spend) already computed
- window functions run AFTER aggregation

So window functions can directly use aggregate results.

==================================================
Why Alias Cannot Be Used
==================================================

This fails:

SELECT
    SUM(spend) AS total_spend,
    ROW_NUMBER() OVER(
        ORDER BY total_spend DESC
    )

Why?

Because:
- aliases are created later
- window functions execute before SELECT aliases exist

==================================================
Easy Memory Trick
==================================================

Window Functions CAN use:
- aggregates
- grouped values

Window Functions CANNOT use:
- aliases created in same SELECT

==================================================
Mental Model
==================================================

GROUP BY creates groups
↓
SUM(spend) calculates totals
↓
ROW_NUMBER ranks totals
↓
SELECT creates aliases
↓
Outer query filters ranks

==================================================
Most Important Interview Pattern
==================================================

Pattern:

SELECT *
FROM (
    SELECT ...,
           ROW_NUMBER() OVER(...)
    FROM ...
) t
WHERE rn <= N

Very common for:
- Top N per category
- Highest salary per department
- Latest order per customer
- Ranking problems
- First login questions
